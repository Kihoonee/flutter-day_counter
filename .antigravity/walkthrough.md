# 알림 설정 개별화(Per-Event Granularity) 작업 결과

사용자의 UX 제안에 따라 전역 설정에 있던 세분화된 알림 옵션을 개별 이벤트 설정으로 이동하여, 각 이벤트의 성격에 맞는 알림을 받을 수 있도록 개선했습니다.

## 주요 변경 사항

### 1. 이벤트별 세부 알림 설정 추가
각 이벤트를 만들거나 수정할 때 다음 알림들을 개별적으로 켜거나 끌 수 있습니다.
- **D-Day 알림**: 당일 오전 9시
- **D-1 알림**: 하루 전 오전 9시
- **기념일 알림**: 100일 단위 기념일
- 알림 메인 스위치를 끄면 하위 세부 옵션들이 숨겨지며 해당 이벤트의 모든 알림이 비활성화됩니다.

### 2. 설정 페이지 단순화
- 설정 메뉴의 '전역 알림 설정'에서는 앱 전체의 알림을 한 번에 제약하는 **마스터 스위치**만 유지했습니다.
- 불필요한 전역 세부 옵션을 제거하여 설정 화면이 더 깔끔해졌습니다.

### 3. 알림 서비스 로직 고도화
- `NotificationService`는 이제 먼저 전역 마스터 스위치를 확인한 후, **개별 이벤트의 알림 옵션**을 하나씩 체크하여 정확하게 스케줄링합니다.

## 검증 결과
- **이벤트 편집 UI**: 개별 이벤트에서 D-Day, D-1, 기념일 알림을 각각 설정할 수 있음을 확인했습니다.
- **설정 UI**: 전역 설정 페이지가 마스터 스위치 하나로 간소화된 것을 확인했습니다.
- **동작 확인**: 전역 알림이 켜져 있어도 개별 이벤트에서 특정 알림(예: D-1)을 끄면 해당 알림만 제외하고 예약되는 것을 확인했습니다.
- **빌드**: iOS 시뮬레이터 및 Android 에뮬레이터에서 정상 작동을 확인했습니다.
